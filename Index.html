How It Works

From diagram to simulation in four steps

System dynamics models describe how stocks accumulate and flows change over time. This builder makes the process intuitive and visual.

1

Define Objects

Create container objects that represent real-world entities — a reservoir, a population, an economy. These become Django models on export.

2

Add Stocks & Constants

Place stocks (accumulators) and constants (fixed parameters) inside objects. Set initial values and units for dimensional analysis.

3

Connect with Flows

Draw flow connections between nodes to define rates of change. Add auxiliary variables and causal links for intermediate calculations.

4

Simulate & Export

Run the simulation with Euler or RK4 integration. Visualize results in real-time charts, then export to Django, JSON, or Vensim MDL.

Building Blocks

The language of system dynamics

Every model is built from a small set of fundamental elements, each with a distinct role.

Stock

Accumulator variable. Changes only through flows.

Flow

Rate of change that transfers between stocks.

Auxiliary

Intermediate calculated value from an equation.

Constant

Fixed parameter that doesn't change over time.

Features

Everything you need to model and simulate

A complete toolkit for system dynamics, designed for researchers and students.

Dual Simulation Engines

Choose between Euler method for quick estimates or 4th-order Runge-Kutta for high-accuracy integration.

Unit Validation

Automatic dimensional analysis checks that your flow units are compatible with their connected stocks.

Django/GeoDjango Export

Generate models.py, views.py, and urls.py with a full simulation API endpoint.

Multiple Export Formats

Save your work as JSON for later, or export to Vensim MDL format for compatibility with other SD tools.

Django Import

Paste an existing models.py and reverse-engineer it into a visual stock-flow diagram.

Real-time Charts

Watch your simulation unfold with interactive charts. Toggle individual variables to focus your analysis.
